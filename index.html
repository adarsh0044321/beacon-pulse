BEACON & PULSE:
Low-Latency LAN Screen Sharing Engine

A high-performance remote desktop streaming and input control pipeline built entirely in Rust. Bypasses standard system copy operations to stream directly from GPU video memories to players at 60 FPS under 1.5ms latency.

Core Capabilities

⚡

Zero-Copy GPU capture

Locks screen surface handles directly inside Windows VRAM, bypassing standard system RAM memory copy cycles entirely.

🖱️

Scan-Code Input Injection

Maps player inputs directly to physical keyboard scan codes. Works layout-independently with absolute zero keystroke delays.

🔒

TLS 1.3 Control channel

Secures keystrokes, clipboard buffers, and file transfer streams over authenticated TLS channels with dynamically rotated local keys.

🌐

📊

Client Telemetry Sync

Measures round-trip time via RTCP probe packet echo loops. Calculates bitrates dynamically and updates real-time player HUD displays.

How It Works

The Data Transmission Loop

Traditional remote desktop protocols are bottlenecked by CPU copying steps. Beacon & Pulse bypasses these layers by linking directly to GPU memory buffers.

1
Windows graphics capture

COM bindings retrieve Direct3D11 surface descriptors directly from the compositor namespace.

2
Hardware Compression

GPU encoders compress textures to H.264 slice streams in under 1.5ms using hardware chips (NVENC/AMF/QSV).

3
RTP packetization & FEC

Frames are packetized into RTP fragments. Parity packets are calculated via XOR blocks to handle UDP network losses immediately.

BEACON & PULSE

Decoupled LAN Remote Control Protocol

Beacon & Pulse is a developer-centric performance system written in Rust, engineered to bypass the latency constraints of traditional remote screen mirroring.

Strategic Advantages

  • Windows Graphics Capture: Locks frame buffers inside GPU VRAM, eliminating expensive copies back and forth to standard system RAM.
  • Hardware H.264 Accelerators: Pulls direct encoding pipelines on physical GPU frames (AMF/NVENC/Intel QSV) yielding sub-millisecond compression.
  • TLS 1.3 Security Handshakes: Encrypts control and keystroke streams via rustls using secure dynamic keys.
