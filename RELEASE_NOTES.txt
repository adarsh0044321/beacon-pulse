📡 Beacon & Pulse v1.1.1 - Release Notes
=======================================

Beacon & Pulse is a low-latency, hardware-accelerated remote desktop screen sharing and control system over local networks built entirely in Rust.

What's New in v1.1.1
-------------------

📱 Android Player Customization & Updates
- Android Player Client Build: The compiled Android Player Client APK is now generated as `PulsePlayer-debug.apk` and is available for download on the release assets page.
- App Icon Customization: The Android app icon is now customized using a high-resolution logo, automatically resized for all standard Android screen densities (mdpi to xxxhdpi).
- Advanced Player Features: Integrated hardware H.264 video decoding using native MediaCodec + SurfaceView for sub-millisecond acceleration, QR camera scan support (zxing-android-embedded) for pairing codes, and high-speed network threading for UDP discovery and TCP control channels.
- Connection Overlay Fix: Resolved sticky "Connecting to host..." overlay by properly propagating the TCP handshake connected status callback event to update UI states.
- Recent Connections Dashboard: Implemented input persistence in SharedPreferences and a scrollable recent connections row (up to 5 items) for instant reconnects.

🌐 Chrome UI Video Stream Fixes
- H.264 Codec Case-Sensitivity: Corrected the profile-level ID to lowercase (`avc1.42c033`) to prevent Chrome WebCodecs from rejecting the video stream with validation errors.
- Secure Context Safeguards: Added client detection for secure contexts (HTTPS/localhost). When accessed over insecure LAN IPs, a descriptive error banner advises users to connect via localhost or enable chrome://flags bypasses.
- Error Handling: Wrapped decoder setup in a try-catch block to gracefully route initialization failures to the user-facing error banner instead of freezing on a black screen.

⚙️ Packaging & Build System Fixes
- Malformed PATH Resolution: Cleaned environment PATH setup in build scripts, bypassing malformed trailing double-quotes in Windows PATH.
- Process Locks: Cleared background process handles to prevent build/copy failures on `beacon.exe` and `pulse.exe`.
- Setup Installer Builds: Compiled self-contained C# setups (`BeaconSetup.exe`, `PulseSetup.exe`) embedding updated Rust binaries and icons.
- Tauri Webview Installer Generation: Generated Chrome UI setups (`BeaconChromeUI-Setup.exe`, `PulseChromeUI-Setup.exe`) using Tauri release compilation and NSIS packaging.

📡 Beacon & Pulse v1.1.0 - Release Notes
=======================================

What's New in v1.1.0
-------------------

🐧 Linux Support & Compilation
- Standalone Linux Binaries: Added compilation support and release packaging for Linux platforms. Linux users can now download `release-linux.zip` containing both the `beacon-host` and `pulse-client` standalone binaries.
- Multi-Platform CI Pipeline: Configured GitHub Actions to build and release both Windows (`release-windows.zip`) and Linux (`release-linux.zip`) standalone bundles.

What's New in v1.0.9
-------------------

🌐 WebSocket IPC Response Resolution Fix
- Promise-Response Matching: Resolved a race condition in Chrome/browser UI mode where background polling requests (like window list/monitor updates) would consume the `start_share` command response. Added strict event-type correlation to ensure the "Start Sharing" action triggers and resolves correctly.

What's New in v1.0.8
-------------------

⚙️ Interactive Installer UI Mode Selection
- Installer UI Prompts: Updated setup installers (`BeaconSetup.exe` and `PulseSetup.exe`) to ask the user what kind of installation/UI mode they prefer:
  - **Localhost Web UI (Chrome UI)**: A user-friendly browser-based interface. Note that this mode cannot hide undetected in the background; the terminal console remains visible, and it operates as a standard foreground application.
  - **Terminal UI / Headless**: Launches as a classic CLI console menu. In this mode, the terminal console window explicitly hides itself, running completely undetected in the background, and is accessible only through the tray overlay menu.
- Shared Configuration: The installer persists this choice as the `UiMode` registry value (`1` for Web UI, `2` for Terminal UI) under `HKCU\Software\Beacon` to control runtime startup behavior.

What's New in v1.0.7
-------------------

🖥️ Minimized Window Interaction Fixes
- Mouse Input Disabling: Skips forwarding mouse movement, scroll, and click coordinates when the target window is minimized, preventing accidental clicks on the host's top-left corner.

🛠️ Offline Discovery & Network Reliability
- Subnet Scan fallbacks: Local subnet discovery fallback now binds to broadcast (`255.255.255.255:53`) and multicast (`224.0.0.1:53`) targets, enabling machine IP resolution completely offline.
- Adaptive Bitrate Overrides: Prevents immediate host rate-controller reset of manual target bitrate adjustments.

📊 Real-Time Network & Client Telemetry
- RTT & Bitrate Syncing: Appends measured round-trip time (`rtt_ms`) to RTCP probe packets, calculates receiver-side bitrate in kbps, and sends the stats over named-pipe IPC to the client viewer HUD.
- Client Debug Overlay (Ctrl+Shift+D): Added client-side stats prop configuration to show real-time FPS, network RTT, decode latency, bitrate, and packet loss, along with a pipeline latency sparkline.
- Key Auto-Release: Clears and dispatches keyup actions for all pressed keys on window blur focus loss.

⚙️ UI Presets & Settings Profiles
- Quick Quality Presets: Added Low Latency (10 Mbps), Balanced (20 Mbps), and High Quality (35 Mbps) quality profile buttons to Settings.
- Bitrate Range Expansion: Extended settings slider max target bitrate limit to 40 Mbps.

What's New in v1.0.6
-------------------

🖥️ Minimized Window Active Sharing & Listing Fixes
- Minimized Window Listing: Restructured window listing filter (`window_list.rs`) to detect minimized windows via `IsIconic` and assign a logical fallback size of `1024x768`. This allows minimized windows (like Chrome, File Explorer, and Antigravity IDE) to be visible and selectable in the console sharing picker across all modes (Single, Multi-Grid, and Dual-Window Side-by-Side).
- Auto-Restoration on Startup: Added auto-restoration checks on startup using `ShowWindow(hwnd, SW_RESTORE)` when sharing a minimized target, ensuring that the window un-minimizes so the OS can render and share its contents.
- Freeze Frame Preservation on Minimize: If a shared window is minimized during the session, the host automatically freezes the stream on the last active un-minimized frame cached in memory. It continues to update timestamps to keep the connection alive and smoothly unfreezes once the window is restored.
- Resolution Collapse Bypass: Prevented `GetWindowRect` from updating captured frame dimensions to the tiny `159x27` minimized title-bar placeholder size when minimized, preserving the last restored resolution.

🛠️ Dynamic Workflows & Automation
- Dynamic CI Release Titles: Configured GitHub Actions workflows (`ci.yml`) to automatically name releases using the git tag (e.g. `Release v1.0.6`) instead of hardcoding past release names.
