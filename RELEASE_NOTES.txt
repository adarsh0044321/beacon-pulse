📡 Beacon & Pulse v1.0.5 - Release Notes
=======================================

Beacon & Pulse is a low-latency, hardware-accelerated remote desktop screen sharing and control system over local networks built entirely in Rust.

What's New in v1.0.5
-------------------

🛠️ Core Latency, Memory, and Stability Fixes
- Windows Media Foundation COM Leak Fix: Implemented an RAII `ActivatesGuard` drop guard to release `IMFActivate` COM handles on exit. This prevents COM leaks on the host during startup or reconfiguration.
- Ring-Buffer Latency Correction: Updated the queue to discard the oldest frame when full instead of the newest, preventing stale frames from creating temporary latency spikes.
- Client Connection Cleanup: Wrapped the client loop in an async block to guarantee that abrupt network drops or reset errors trigger client session deregistration.

🖥️ Front-End & Input Synchronization
- Settings Registry Synchronization: Connected Tauri front-end settings directly to the Windows Registry (`HKCU\Software\Beacon`). Setting changes (bitrate, FPS, control permissions, and startup rules) are immediately visible to the background core service and watchdog.
- Mouse Aspect-Ratio Calibration: Calculated the active video viewport within the player canvas to discard letterbox/pillarbox margins, scaling mouse coordinates correctly relative to actual video bounds.
- Layout-Independent Keyboard Lookup: Mapped JS `KeyboardEvent.code` directly to Windows scan codes and extended key flags (`is_extended`) to handle non-US layouts and arrow/navigation keys.
- Verification Testing: Added automated unit tests verifying the lock-free ring-buffer queue behavior.
