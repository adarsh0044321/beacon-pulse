Engine Architecture

Zero-Copy GPU Surface Capture

Locks framing textures in local VRAM via Windows Graphics Capture COM handles. Prevents pipeline overhead by completely skipping system memory copies back to CPU RAM.

Scan-Code Key Injection

Sends keystrokes as physical driver scan codes. Works layout-independently on local devices with absolute zero processing delays.

Subnet Auto-Discovery

Discovers screen sharing streams via asynchronous browse threads, scanning local networks dynamically with mDNS and parallel UDP socket queries.

Secured TLS 1.3 Control Pipeline

Encrypts active input vectors, file transfers, and clipboard buffers over authenticated channels. Rotates keys locally on launch using a secure cryptographic setup.

Execution Cycle

VRAM-to-Display Pipeline

Standard sharing tools delay signals using system RAM copies. Beacon & Pulse links display buffers directly inside GPU memory to achieve microsecond-range rendering.

1
Ingest surface buffers

Retrieves surface descriptors directly from active Windows compositor threads.

2
Hardware Compression

Compresses video textures to H.264 slice units on hardware media cores (NVENC/AMF).

3
RTP stream packaging

Splits slices to UDP network packets, adding parity codes for instantaneous client packet rebuilding.

Beacon & Pulse

Decoupled LAN Remote Control Pipeline

Beacon & Pulse is a low-latency screen mirroring platform written entirely in Rust, designed to support quick local desktop control feeds.

Key Features

  • GPU VRAM Capture: Pulls desktop frames directly from Windows compositor surfaces inside graphics memory.
  • Hardware Compressors: Utilizes physical GPU cores (AMF/NVENC/Intel QSV) for ultra-low latency texture encoding.
  • Secure IPC Channels: Encrypts file transfer packets and mouse cursor arrays over strict TLS 1.3 socket streams.
